WebCrypto GOST Class: GostPKIX

GostPKIX

Constructor

new GostPKIX(providerName, outputFormat, keyStore)

PKIX public class

Supported providers:
  • CP-94 - CryptoPro GOST R 34.10-94 algorithm set
  • CP-01 - CryptoPro GOST R 34.10-2001 algorithm set
  • TC-256 - Technical Commitee GOST R 34.10-256 algorithm set
  • TC-512 - Technical Commitee GOST R 34.10-512 algorithm set
  • SC-94 - SignalCom GOST R 34.10-94 algorithm set
  • SC-01 - SignalCom GOST R 34.10-2001 algorithm set
Follow set can be used if it's supported your browser native WebCrypto API:
  • RSA-1024 - RSA Encryption 1024 bits with SHA-1 algorithm set
  • RSA-2048 - RSA Encryption 2048 bits with SHA-256 algorithm set
  • RSA-4096 - RSA Encryption 4096 bits with SHA-512 algorithm set
  • ECDSA-256 - ECDSA-256 with SHA-256 algorithm set
  • ECDSA-384 - ECDSA-384 with SHA-384 algorithm set
  • ECDSA-521 - ECDSA-521 with SHA-512 algorithm set
Supported output formats:
  • DER - DER encoded binary ArrayBuffer
  • PEM - PEM encoded string
For storage of keys, you can use an external repository, if you implement the KeyStore interface .
Parameters:
Name Type Description
providerName string Name of crypto provider, defined in gostObject.providers
outputFormat string Format for output encoded data: DER or PEM
keyStore KeyStore Optional external key store. Internal storage used if the keyStore not specified. Storage objects like localStorage, sessionStorage with functions setItem and getItem also can be used.

encryptData(data, mode, recipient, alias, password) → {Promise}

Encrypt data using KeyTransport or KeyAggrement algorithm in CMS format

Follow modes supported:
  • 'keyagree' Key agreement protocol
    • encryptData(data, 'keyagree', recipient) - generate ephemeral keys
    • encryptData(data, 'keyagree', recipient, alias, password) - use sender key
  • 'keytrans' Key transport protocol
    • encryptData(data, 'keytrans', recipient) - generate ephemeral keys
    • encryptData(data, 'keytrans', recipient, alias, password) - use sender key
  • 'kek' Key encryption key protocol
    • encryptData(data, 'kek', alias, password) - use secret key from key store
  • 'pbkek' Password based key encryption key protocol. Result EnvelopedData
    • encryptData(data, 'pbkek', password) - use password for key encryption
  • 'keyman' Key management protocol. Simple encryption with symmetric key
    • encryptData(data, 'keyman', alias, password) - use secret key from key store
  • 'pbes' Password based encryption. Key produced direct from password
    • encryptData(data, 'pbes', password) - use password for generate key
Parameters:
Name Type Description
data FormatedData Source of data
mode strings Encryption mode: 'keyagree', 'keytrans', 'kek' or 'pbkek', 'keyman', 'pbes'. default 'keyagree'
recipient string | Array.<string> Recipient alias or aliases. Parameter must be ommited for 'pbkek', 'kek', 'keyman', 'pbes' modes.
alias string Optional key alias. Specify if the key of sender uses. Parameter must be ommited 'pbkek' mode.
password string Optional password. Specify if the key of sender uses and in password mode. Parameter must be ommited for 'kek' mode.

importKey(key, alias, password) → {Promise}

Import key to keystore

Autorecognized and supported formats:
  • raw Raw binary format
  • p8 PKCS#8 PrivateKeyInfo format
  • p8e PKCS#8 EncryptedPrivateKeyInfo format
  • p12 PFX key store format
If password (or secret key) specified key stored in encryption mode. If key already encrypted or password (or secret key) not specified key stored w/o any conversion. For change key password (or secret key) simple exportKey in 'p8' format with old password (or secret key) and importKey with new password (or secret key).
